## Featured Case Studies

### Bong Pizza Keshapat — WhatsApp-Native Ordering System

A pizza and fast-food restaurant in rural West Bengal, outside the reach of Swiggy or Zomato. The challenge: build a real ordering experience without a backend, without a platform commission, and without anything the restaurant would have to maintain.

**The approach:** A 61-item, 10-category cart-first menu built in React + TypeScript. Persistent cart state. Delivery-zone checking. WhatsApp checkout — a formatted order summary handed off to staff directly, where they confirm availability and pricing in a conversation the restaurant already knows how to have.

> *Aggregator commissions run 20–30%. This site costs the restaurant nothing per order.*

[↗ Live Site](https://bong-pizza-renew.vercel.app) · [↗ Case Study](https://portfolio-v2-tau-eosin.vercel.app/projects/bong-pizza-keshapat)

---

### Café Elio — Multi-Page Experience for a Quirky Kolkata Hangout

A pool-table-and-graffiti-wall café in Jadavpur whose reviews talk about the room as much as the food. Generic café websites lead with a hero banner and a menu. This one leads with the space.

**The approach:** A 10-page React site (Home, Menu, About, Gallery, Reservation, Events, Journal, FAQ) where the gallery, atmosphere, and character of the room are first-class content — not an afterthought after the menu. Reservation system with validation, confirmation state, and large-party redirect. Intro splash on first visit via sessionStorage.

[↗ Live Site](https://cafe-elio.vercel.app) · [↗ Case Study](https://portfolio-v2-tau-eosin.vercel.app/projects/cafe-elio)

---

### Finca's Coffee — Farm-to-Cup Brand Experience in Vanilla JS

One family. One farm in Honduras. Four generations. One café in Miami. A supply chain that's a better story than most brands' entire About pages.

**The approach:** A deliberately vanilla JS build — no framework, maximum creative control, minimum overhead. The page is structured as the supply chain itself: farm origin → harvest timeline → roast → pour → visit. Dynamic accent-color theming (Olive, Copper, Espresso, Terracotta) tied to drink categories. Parallax depth on key sections. Fluid `clamp()` typography from 320px to 4K.

[↗ Live Site](https://finca-coffee.vercel.app) · [↗ Case Study](https://portfolio-v2-tau-eosin.vercel.app/projects/finca-coffee)

---

### LNJ Billing — Jewellery Shop Management System

Manual billing for jewellery retail means every making charge and GST split gets hand-calculated, and inventory is a separate ledger that silently drifts from what's actually in the showcase — discovered only at audit time.

**The approach:** A product-ID-centric system where every invoice line, stock movement, and barcode resolves back to one canonical product record. Bills are priced from a live gold rate plus making charge. Stock deducts atomically on every sale. 14 feature modules: billing, invoicing, inventory, stock movements, barcode generation, CRM, analytics, calendar, product management, PDF export, Excel import/export, reports, rates, settings.

[↗ Live Site](https://implement-of-login-page.vercel.app) · [↗ Case Study](https://portfolio-v2-tau-eosin.vercel.app/projects/lnj-billing)

## Design Language

This portfolio and all client work share a deliberate design philosophy:

```
Editorial, not decorative.
Restrained, not minimal.
Precise, not cold.
```

**Typography** — Instrument Serif for display, Space Grotesk for body. Serif carries the emotional weight; sans carries the information. They do not compete.

**Color** — Cream ground (`#f3eee3`), ink text (`#15130f`), dark red accent (`#7a1f1f`). Three colors. No more.

**Motion** — Reveal animations on scroll, word-level text reveals, staggered grid entrances. Everything timed to `cubic-bezier(0.16, 1, 0.3, 1)` — fast start, long tail, editorial weight.

**Whitespace** — Structural, not decorative. Whitespace is the clearest signal of confidence in the content.

## Portfolio Architecture

```
src/
├── components/
│   ├── layout/          # Navigation, Footer
│   ├── sections/        # Homepage section components
│   ├── ui/              # Button, Reveal, FAQAccordion, FeatureGrid, EditorialList
│   └── work/            # ProjectSection, EditorialCTA, AnimatedText
├── constants/
│   └── nav.ts           # Single source of truth for all navigation
├── data/
│   ├── projects.ts      # All 17 case studies — one object per project
│   ├── services.ts      # Services page content
│   ├── process.ts       # Process page content
│   └── contact.ts       # Contact page content
├── pages/
│   ├── Home.tsx
│   ├── Projects.tsx     # Editorial project index
│   ├── ProjectDetails.tsx   # Dynamic case study page (/projects/:slug)
│   ├── Services.tsx
│   ├── Process.tsx
│   └── Contact.tsx
└── types/
    └── content.ts       # Shared FAQ, Feature types
```

**Adding a new project** = adding one object to `src/data/projects.ts`. The index page, homepage featured section, and full case study page derive from it automatically.

## Local Development

```bash
# Clone the repository
git clone https://github.com/ShibamPandab/Portfolio_V2.git
cd Portfolio_V2

# Install dependencies
npm install

# Start the development server
npm run dev
# Runs at http://localhost:5173

# Build for production
npm run build

# Preview the production build
npm run preview
```

**Requirements:** Node.js 20+
