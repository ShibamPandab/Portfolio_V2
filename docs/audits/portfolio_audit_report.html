P0 — Must Fix Before This Ships

P0 · Work section has zero actual work

Every single case study shows "VISUAL PENDING" in the preview slot. You're applying for a frontend role and your portfolio has no screenshots, no live links, no visual proof of execution. This is the single most damaging thing on the page. A hiring manager at Vercel or Linear closes this tab the moment they see that placeholder repeated 5+ times.

→ Ship with 3 real projects with real screenshots. Fewer real entries beat more placeholder ones every time. You have 11 deployed projects — pick your 3 strongest and add actual images today.
P0 · Duplicate CTA creates conversion paralysis

The nav has "START A PROJECT" (top right), the hero has "START A PROJECT →" as a ghost link AND "VIEW CASE STUDIES" as primary — and the footer repeats it again. Two competing primary actions in the hero means neither is primary. The black button says one thing; the ghost link next to it says another. The visitor's eye doesn't know where to go.

→ Hero: one primary CTA — "View My Work" (black button). Remove the ghost "START A PROJECT" from the hero entirely. Let the nav CTA handle the hire intent. Keep the footer CTA as the closer.
P0 · Stats section is factually embarrassing for a senior candidate

The "Selected Numbers" shows 3 business categories, 0 templates used, 100% responsive. These are not the numbers a senior developer leads with. "3 categories designed for" is thin. "0 templates" is a negative claim dressed up as a positive. "100% responsive" is table stakes — it's like a chef listing "100% cooked food" as a credential.

→ Replace with real proof: number of deployed projects, client satisfaction metric (even NPS), or Lighthouse scores from your actual builds. If you have 11 projects live, say "11 production deployments." That's real.
P0 · Email is plain text — not a link

shibampandab@gmail.com in the footer is an unlinked string. Clicking it does nothing. On mobile it's not even tappable. This is a broken contact path on a client-acquisition portfolio.

→ Wrap in <a href="mailto:shibampandab@gmail.com">. Also: a Gmail address signals student project, not freelance professional. Consider a custom domain email.

P1 — Fix Before Showing to Clients or Recruiters

P1 · "00" eyebrow is structurally incoherent

The hero uses "00" as a section number. Your subsequent sections use 01, 02, 03, 04, 05. Section 00 would imply a prologue or intro — but the hero is actually the primary message, not a preface. This numbering device signals "I copied an editorial convention without understanding why it exists." On a portfolio for a designer-developer, that's a meaningful tell.

→ Remove "00" entirely. The hero doesn't need a section number — it's the thesis, not a chapter. Start numbering at 01 for the first content section.
P1 · The italic accent word is a repeated trick, not a system

The crimson italic customers in the hero headline is your signature move. Then in the philosophy section, crimson italic remembered appears again — same typeface, same color, same treatment, same mid-sentence position. Then there's a pullquote using the same italic face in full block size. Three uses of the same trick in one scroll = it stops being a choice and starts looking like a template.

→ Use the crimson italic in exactly one place — the hero. Make it singular. The pullquote is strong enough on its own in the serif body face without the crimson.
P1 · The 4-column features band has weak copy

"Design-Led," "Built to Convert," "AI-Accelerated," "Production-Ready" — these are the 2024 freelance web dev value-prop defaults. Every Framer template, every Webflow agency, every Fiverr seller uses at least two of these phrases. "AI-Accelerated" as a selling point to a premium client is a red flag, not a benefit — it implies speed over craft. "Production-Ready" is what all code should be.

→ Rewrite from the client's perspective. What does the client gain? "Bookings before the site launches" beats "Built to Convert." "Ships in 2 weeks, not 2 months" beats "AI-Accelerated." Make it specific and time-bound.
P1 · Case study copy is self-referential, not client-outcome-oriented

The Bong Pizza write-up says the old site had "a stock hero photo and a menu buried in a PDF." That's a diagnosis. There's no outcome stated anywhere — no increase in online orders, no reduction in support calls, no measurable result. The quote "The food had more personality than the website did" is styled as a client testimonial but reads like you wrote it. There's no attribution — no name, no role, no business.

→ Add real outcomes: "Online orders up 40% in first month." If you don't have the numbers yet, get them from the client. If you can't, at least attribute the quote: "— [Name], Owner, Bong Pizza."
P1 · Hero subline buries the lead

The subtext reads: "Premium, design-led websites for restaurants, boutiques, and brands that refuse to look ordinary online — built fast with an AI-accelerated workflow, finished by hand." The strongest claim — "finished by hand" — is at the end, buried after a long em-dash clause. "AI-accelerated workflow" in the middle undercuts the craft message you're selling. The sentence tries to say three things and lands none of them cleanly.

→ Cut it in half: "Premium websites for restaurants, boutiques, and brands — built by hand, delivered fast." Lead with the client outcome, not your process.

P2 — Polish Gaps

P2 · Vertical scroll cue "SCROLL" text on the features band

The rotated "SCROLL" label on the right edge of the 4-column features section is decoration with no function. The section is already visible. The user doesn't need to be told to scroll — they're already scrolling. This is a visual tic borrowed from agency sites where it made more sense.

→ Remove it. The page is better without it.
P2 · The "Good design isn't decoration" section is orphaned

A large full-width italic serif pullquote appears between the features strip and the philosophy section with a big empty gap above it. It floats without structural anchor — no section label, no connection to what came before. It reads as filler between two real sections.

→ Either integrate it as the opening statement of the philosophy section (with its eyebrow above it), or cut it. The philosophy section already has a strong pullquote ("A website is the first conversation…") — you don't need two.
P2 · No social proof for a service business

There are no client names, no testimonials with attribution, no logos, no GitHub link, no LinkedIn. For a freelancer targeting international clients, social proof is the trust signal that converts. The quoted "client" copy reads invented because it has no anchor.

→ Add one real testimonial from a real client with name + business, or a GitHub link showing real commit history. Either grounds the portfolio in reality.

What's Actually Working

The typographic scale in the hero is legitimately commanding. The headline at ~100px with the mixed-weight contrast (light sans + crimson italic serif) is the strongest moment on the page — it has visual authority and personality. The cream/black palette is restrained and coherent. The case study copy voice (before the attribution problem) is more writerly than 95% of freelance portfolios. The pullquote "A website is the first conversation most customers have with a business" is genuinely good — it earns its full-screen placement.

Recommended First Action

Single most impactful change

Add real screenshots to 3 case studies this week. Nothing else on this list matters until the work section has actual work in it. You have 11 deployed projects — screenshot them tonight, put the 3 best in. A portfolio with 3 real visible projects and zero placeholder boxes outperforms this current page by an order of magnitude for any hiring manager or international client.
